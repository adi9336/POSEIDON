1. Primary Streamlit Execution Flow

Runtime Chain

Streamlit Entry

streamlit_app.py -> app.py initializes runtime and UI state.

Supervisor Gate

src/supervisor/agent.py decides: clarify, map confirm, approval, or execute.

Orchestrator Graph

src/orchestrator/main.py builds and runs workflow graph with events.

Agent Pipeline

Understanding -> Retrieval -> Analysis -> Validation in src/agents/.

Result + Visuals

Diagnostics + Plotly/Folium rendering in app.py, plus final response.

2. Agent Stage Internals

Query Understanding

Parses natural-language request into structured scientific intent and requested variables.

src/agents/query_understanding/agent.py

Data Retrieval

Resolves location/time/depth and fetches Argo data via ERDDAP into runtime data snapshots.

src/agents/data_retrieval/agent.py + src/tools/fetcher.py

Analysis

Processes data through SQL/scientific logic and composes summary + variable insights.

src/agents/analysis/agent.py + src/tools/processor.py

Validation

Applies scientific plausibility checks, outlier-rate checks, and confidence scoring.

src/agents/validation/agent.py

3. Data + Memory Flow

Raw Data Ingest

Fetched observations saved in data/ and loaded into data/argo_data.db.

Scientific Processing

SQL-backed analysis and derived metrics from toolchain in src/tools/.

Context Retrieval

src/memory/context_builder.py enriches orchestrator with historical context.

Insight Write-back

insight_store.py stores structured records; semantic vectors are optional.

Confidence-Aware Output

Final response includes validation report, confidence, and execution metadata.

4. Alternate Workflows

FastAPI Path

src/api/server.py serves /v1/query (orchestrator), websocket event stream, and compatibility /query.

Legacy Compatibility Path

Legacy graph in src/agent/ remains for fallback/compat mode and transition safety.

Campaign Automation Path

src/campaigns/manager.py schedules anomaly watch, weekly reports, and float watcher.

Controlled by POSEIDON_ENABLE_CAMPAIGNS.
