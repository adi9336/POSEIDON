POSEIDON Full Project Workflow

This page explains the end-to-end runtime architecture of POSEIDON, from Streamlit input to validated oceanographic output.

1. Deployment Entry Points

UI uses streamlit_app.py (delegates to app.py). API uses src/api/server.py.

Streamlit: streamlit_app.py UI Logic: app.py API: src/api/server.py Runtime Config: .streamlit/config.toml Python Runtime: runtime.txt

2. Primary Request Pipeline

User Query

User asks a natural-language ocean data question in Streamlit/API.

Supervisor Layer

src/supervisor decides clarify/map/approval/execute.

Orchestrator

src/orchestrator/main.py runs graph and emits events.

Agent Chain

Understanding -> Retrieval -> Analysis -> Validation.

Response + Memory

Return diagnostics/plots and persist insight context.

User -> Supervisor -> Orchestrator -> Agents -> Output

3. Orchestrator Stage Details

Understanding

Extracts structured intent and requested variables from query text.

Module: src/agents/query_understanding/agent.py

Retrieval

Fetches ERDDAP Argo data and stores snapshots into data/.

Modules: src/agents/data_retrieval/agent.py, src/tools/fetcher.py

Analysis

Runs SQL + scientific processing and creates summaries/insights.

Modules: src/agents/analysis/agent.py, src/tools/processor.py

Validation

Checks plausibility ranges, outlier rates, time-window consistency, and confidence.

Module: src/agents/validation/agent.py

4. Supervisor Decision Workflow

The Supervisor is the conversation controller before expensive execution:

  1. Parse and reason over user intent.
  2. If critical parameters are missing, ask clarification questions.
  3. If location is present, request map confirmation and area radius.
  4. Build an approval plan (time/cost/data estimates).
  5. On approval, trigger orchestrator execution.

Modules: src/supervisor/agent.py, state.py, clarification.py, map_interface.py, planner.py.

5. Data, State, and Memory Architecture

Operational Data

data/argo_data.db is the primary runtime SQLite store used by processor/API health/map coverage checks.

Memory Layer

src/memory builds baseline/trend context and stores insights for future enrichment.

State Contracts

src/state/models.py and src/state/schemas.py define intent/task/result/event contracts.

Core Paths

src/core/paths.py centralizes directories and runtime paths.

6. Secondary Workflows

Legacy Compatibility Path

src/agent still supports legacy execution mode and compatibility API behavior.

Campaign Automation

src/campaigns can run scheduled anomaly/weekly/float monitoring jobs. This is disabled by default for cloud Streamlit via POSEIDON_ENABLE_CAMPAIGNS=false.

7. Production Notes

  • Use requirements.txt for production deployment; use requirements-dev.txt only for local development.
  • Set OPENAI_API_KEY in Streamlit secrets/environment.
  • Keep heavy semantic memory features off by default unless compute/memory budget is sufficient.
